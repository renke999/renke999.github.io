    TEX 是高德纳 (Donald E.Knuth) 开发的、以排版文字和数学公式为目的的一个计算机软件

    LATEX 为 TEX 基础上的一套格式，令作者能够使用预定义的专业格式以较高质量排版和印
    刷他们的作品。

    latex 虽然名为 latex 命令，底层调用的引擎其实是 pdfTEX。该命令生成 dvi（Device Independent）格式的文档，用 dvipdfmx 命令可以将其转为 pdf。

    xelatex 底层调用的引擎是 XLTETEX，支持 UTF-8 编码和 TrueType / OpenType 字体。当前较
    为方便的中文排版解决方案基于 xelatex

    使用xelatex test.tex编译可直接生成.pdf文件

  • 为了编译方便同时删除中间文件可以新建一个批处理文件 compile.bat

2. 源文件的基本结构

  • 分为导言区和正文区

    • LATEX 源代码以一个\documentclass命令作为开头，它规定了文档使用的文档类：

      article 文章格式的文档类，广泛用于科技论文、报告、说明文档等
      report 长篇报告格式的文档类，具有章节结构，用于综述、长篇论文、简单的书籍等
      book 书籍文档类，包含章节结构和前言、正文、后记等结构
      option部分：
      10pt, 11pt, 12pt 指定文档的基本字号。缺省为 10pt。 > a4paper, letterpaper, … 指定纸张大小，缺省为美式信纸 letterpaper（8.5×11 英寸）
      twoside, oneside 指定单面/双面排版。
      onecolumn, twocolumn 指定单栏/双栏排版。缺省为 onecolumn

    • 之后用\usepackage命令调用宏包：

    • 再接着，用document环境来标记正文内容范围：

    • 在\documentclass和\begin{document}之间的位置称为导言区，除了使用 \usepackage
      调用宏包之外，一些对文档的全局设置命令也在这里使用。

LATEX 引入了环境的用法，用以令一些效果在局部生效，或是生成特殊的文档元素。 LATEX
环境的用法为一对命令 \begin 和 \end：
\begin{⟨environment name⟩}{⟨arguments⟩}
. . .
\end{⟨environment name⟩}

  • 部分文件说明

    • .log 排版引擎生成的日志文件，供排查错误使用。
    • .aux LATEX 生成的主辅助文件，记录交叉引用、目录、参考文献的引用等。
    • .sty 宏包文件。宏包的名称与文件名一致。
    • .cls 文档类文件。文档类名称与文件名一致。

3. 中文处理办法

xelatex 命令支持直接使用系统安装的 TrueType (.ttf) / OpenType (.otf) 等格式的字
体，加上对 UTF-8 编码的原生支持，免去了预处理字体的麻烦。在此基础上的 xeCJK 宏包更进
一步完善了排版中文的一些细节，比如中英文之间插入空隙、中文行尾的回车不引入空格、标点
符号不出现在行首，等等 。而ctex 宏包和文档类是对 CJK 和 xeCJK 等宏包的进一步封装。 ctex 文档类包括 ctexart / ctexrep / ctexbook，是对 LATEX 的三个标准文档类的封装，对 LATEX 的排版样式做了许多调整，
以切合中文排版风格。

详细内容见手册ctex,lshort-zh

  1. 设置编译器为XELATEX，编码为utf-8

  2. \documentclass{ctexart}

或者
\documentclass{article}

5. 篇章结构

  • 标题页：

    给定标题作者等信息：\title{⟨title⟩}，\author{⟨author⟩}，\date{⟨date⟩}

    在合适的位置使用\maketitle

  • 篇章部分：

    使用\section{⟨title⟩}，\subsection{⟨title⟩}，\subsubsection{⟨title⟩}，\chapter进行划分章节。其中\chapter只在 book 和 report 文档类有定义。这些命令生成章节标题，并能够自动编号。

    参数设置

    • 带可选参数的变体：\section[⟨short title⟩]{⟨title⟩}
      标题使用 ⟨title⟩ 参数，在目录和页眉页脚中使用 ⟨short title⟩ 参数；
    • 带星号的变体： \section*{⟨title⟩}
      标题不带编号，也不生成目录项和页眉页脚。

    在ctexart中的一级标题是居中的，可以查看ctex手册使用

  • 目录

    在合适的位置使用\tableofcontents即可

7. 插图

LATEX 本身不支持插图功能，需要由 graphicx 宏包辅助支持。

详细查看手册 texdoc graphicx

  • 导入宏包

    • 导入graphicx宏包
    • graphicx 宏包还提供了\graphicspath 命令，用于声明一个或多个图片文件存放的目
      录，使用这些目录里的图片时可不用写路径

  • 加载图片

    • 使用命令\includegraphics[⟨options⟩]{⟨filename⟩}

    • 可使用图片的格式：ESP, PDF, PNG, JPG, BMP

    • 其中 ⟨options⟩ 为可选参数

      参数 含义
      width=⟨width⟩ 将图片缩放到宽度为 ⟨width⟩
      height=⟨height⟩ 将图片缩放到高度为 ⟨height⟩
      scale=⟨scale⟩ 将图片相对于原尺寸缩放 ⟨scale⟩ 倍
      angle=⟨angle⟩ 令图片逆时针旋转 ⟨angle⟩ 度

    • 其中 ⟨filename⟩ 为图片文件名，与使用\include 命令类似（命令 \include用来在源代码里插入文件：
      \include{⟨filename⟩}），文件名有时需要使用相对路径或绝对路径。图片文件的扩展名可写可不写。

9. 浮动体

往往一些图片和表格等内容尺寸太大，导致分页困难。 LATEX 为此引入了浮动体的机制，令大块的内容可以脱离上下文，放置在合适的位置。
LATEX 预定义了两类浮动体环境 figure 和 table。习惯上 figure 里放图片， table 里放表格，但并没有严格限制，可以在任何一个浮动体里放置文字、公式、表格、图片等等任意内容。

作用：实现灵活分页（避免无法分割的内容产生的页面留白）、给图表增加标题、交叉引用（交叉引用所有编码非硬编码，是自动生成的）

所有宏包可通过texdoc+宏包名进行手册的查询

  • 标题控制用caption, bicaption等宏包
  • 并排与子图表用subcaption, subfig等宏包
  • 绕排用picinpar, wrapfig等宏包

  • 用法：

    <允许位置参数> (默认tbp)

    • h：此处（here），代码所在的上下文位置
    • t：页顶（top），代码所在页面或之后页面的顶部
    • b：页底（bottom），代码所在页面或之后的底部
    • p：独立一页（page），浮动页面

    • \centering：居中排版

    • \caption：标题

    • 交叉引用：使用\label{text}设置标签，使用\ref{text}进行引用

      \label命令可用于记录各种类型的交叉引用，使用位置分别为：
      章节标题：在章节标题命令 \section 等之后紧接着使用。
      行间公式：单行公式在公式内任意位置使用；多行公式在每一行公式的任意位置使用。
      图表标题：在图表标题命令 \caption 之后紧接着使用。

12. 数学公式中多行公式

LATEX 提供了eqnarray环境。它按照等号左边——等号——等号右边呈三列对齐，但等号周围的空隙过大，加上公式编号等一些 bug，目前已不推荐使用。

通过导入amsmath宏包使用gather，align等环境。

  • gather环境用于罗列多个公式，gather*用于不带编号的公式。

    $\begin{gather} % 实现分行排版并编号
    a + b = b + a \notag \ % 可使用notag不编号
    ab = ba\
    (ab)c = a(bc)
    \end{gather} $

  • align环境将公式用 & 隔为两部分并对齐。分隔符通常放在等号左边，align*用于不带编号公式。

    $\begin{align} % 用&对其

    x &= 7y + 9\\
13y &= 3x + 5

    \end{align}$

  • split可将多行定义为同一个公式，即用于一个公式需要多行排版时可采用align环境的对齐方式。

    $ \begin{equation}
    \begin{split}

    \cos 2x &= \cos^2 2x - \sin^2 2x \\
&= 2\cos^2 2x - 1

    \end{split}
    \end{equation}$

  • case环境，每行公式使用&分割为两部分。

    \text用于临时切换中文模式。

    \mathbb用于实现花体英文$\mathbb{R}$，需要导入amssymb宏集。

    $\begin{equation}
    D(x) =
    \begin{cases}

    1, & \text{如果 } x \in \mathbb Q; \\
0, & \text{如果 } x \in \mathbb{R}\setminus\mathbb{Q}.

    \end{cases}
    \end{equation}$

13-14. 参考文献BibTex

LATEX 提供的参考文献和引用方式比较原始，而BIBTEX 是最为流行的参考文献数据组织格式之一。它的出现让我们摆脱手写参考文献条目的麻烦。 BIBTEX 数据库以 .bib 作为扩展名，其内容是若干个文献条目。详情可查看参考文献

多数时候，我们无需自己手写 BIBTEX 文献条目。从 Google Scholar 或者期刊/数据库的网站上都能够导出 BIBTEX 文献条目或者可以使用Zotero。

可查看此文 https://www.jianshu.com/p/0f0eeb9e96ac

  • 使用方法

    • 在源代码内（一般在导言区）使用 \bibliographystyle 命令：\bibliographystyle{⟨bst-name⟩}

    • 整体操作流程（from lshort-zh）

      1. 准备一份 BIBTEX 数据库，假设数据库文件名为 books.bib，和 LATEX 源代码一般
        位于同一个目录下。

      2. 在源代码中添加必要的命令。假设源代码名为 demo.tex。

        1. 首先需要使用命令 \bibliographystyle 设定参考文献的格式。

        2. 其次，在正文中引用参考文献。 BIBTEX 程序在生成参考文献列表的时候，通常只列出用了
          \cite 命令引用的那些。如果需要列出未被引用的文献，则需要 \nocite{⟨citation⟩} 命令； \nocite{*} 则让所有未被引用的文献都列出。

  1. 再次，在需要列出参考文献的位置，使用 \bibliography 命令\bibliography{⟨bib-name⟩}

    其中 ⟨bib-name⟩ 是 BIBTEX 数据库的文件名， 不要带 .bib 扩展名。

    注意： \bibliographystyle 和 \bibliography 命令缺一不可，没有这两个命令，使用BIBTEX 生成参考文献列表的时候会报错。

  2. 编译（具体用编译器即可，详细见文档）

15 自定义命令和环境

  • 定义新命令

    • 具体格式：\newcommand{\⟨name⟩}[⟨num⟩]{⟨definition⟩}

    • 第一个name 是你想要建立的命令的名称（带反斜杠），第二
      个definition 是命令的定义。方括号里的参数num是可选的，用于指定新命令所需的参数数
      目（最多 9 个）。如果缺省可选参数，默认就是 0，也就是新建的命令不带任何参数

    • 带参数的命令类似于函数传参

  • 定义新环境

    • 具体格式：\newenvironment{⟨name⟩}[⟨num⟩]{⟨before⟩}{⟨after⟩}

    • \newenvironment 命令有一个可选的参数。在 before中的内容将在此环境包含
      的文本之前处理，而在 after中的内容将在遇到\end{⟨name⟩}命令时处理。
